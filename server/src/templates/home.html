Wireframe of IA Design:

Firstly the users will be introduced to the homepage which will look fairly straight forward. We have already developed a basic login page that will hold credentials in a database. The reason for this is that each individual that uses our product will have their own "faces" that will be recognized.

Once logged in, we plan to have a simple interface with a few buttons similar to the Tello Drone controller.

We will have an area for the camera feed (strictly for taking a photo and then displaying it, not a live feed) and buttons to add this photo to the list of recognized faces. Since this area will be empty in the beginning (no photo = no display) we plan to have simple instructions demonstrating how to set up accounts and users.

There will be a text list of users that are associated with the account, basically people who are able to operate the door. Adding new users will have 2 steps : taking the picture such that it satisifies the user, and adding the picture to the database. Editing users will redirect users to a new page in which they can choose which entries to delete or change the names. Changing the photo associated with that name may be a possible implementation.

Whenever users want to operate the door or simply test it, they can click scan face which will run the proper procedures to compare the current face in front of the webcam to the pictures stored in the database.

Here is a basic layout that will most likely be changed in the future

Once logged in users will have the option to log out (will be added later). There is only currently 2 routes for users to directly access, which is the login page and the actual app interface, which is called index.html.

Using simple buttons seemed like the most logical choice since every option we have on our interface is simply yes or no, so there was not much need for special checkboxes or radio buttons.
